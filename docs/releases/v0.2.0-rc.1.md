# Hyprland Settings v0.2.0-rc.1 (release candidate)

This is a **release candidate**, not the final v0.2.0. It exists so the
runtime-first preview and gated persistence work can be exercised as a
packaged build before the user approves the final release.

## Release scope

Runtime-first live preview and gated single-write persistence for the
v0.55.2 data/model, audited at zero drift against Hyprland 0.55.4.

## What is new since v0.1.0

- Runtime preview capability system covering all 341 scalar rows with
  honest per-row classifications: 135 rows preview live (62 direct, 73
  throttled), 78 run under dead-man supervision, the rest are blocked with
  reasons.
- Dead-man supervised preview: countdown auto-revert unless kept; 36
  input/cursor rows promoted by live proofs on real hardware.
- Safe Live Save Mode: the app disables config autoreload at runtime (no
  file write, instantly reversible) so saving cannot reload the compositor.
  Every production Save verifies the mode live first.
- Gated Save: byte-exact backup, one atomic write, reread verification
  through the parser, automatic backup restore on verification failure.
- "Save as default" for Safe Live Save Mode (`misc:disable_autoreload =
  true`) through the same gated Save — user-chosen, never automatic.
- Structured-family record picker: modify existing animation records
  (enabled flag, speed, bezier reference — existing curves only) and bezier
  curves (all four control points), with supervised preview and gated Save.
  Every editable field shape carries a passed live-proof receipt; the style
  field is deliberately not editable (no trusted value evidence) and is
  preserved on save. Creating and deleting records stays blocked.
- Hyprland 0.55.4 compatibility: zero drift against a trusted
  `hyprctl -j descriptions` export, with a repeatable read-only refresh
  workflow (`tools/refresh_hyprland_descriptions_export.sh`).

## What is intentionally not included

- Record creation and deletion (blocked by construction).
- Editing monitor/bind/gesture/device/permission families.
- The animation style field (no trusted evidence for valid values).
- 18 touch-family rows and 3 secondary-device rows (deferred pending real
  hardware; disarmed, not broken).
- The final v0.2.0 tag, GitHub release, and any published artifacts.

## Safety notes

- Normal `cargo test` never writes the active config and never mutates the
  running compositor; live proofs are env-gated, `#[ignore]`d, backup-first,
  and restore byte-exactly.
- The packaged app writes the active config only through the gated Save,
  which requires Safe Live Save Mode to be active at runtime.
