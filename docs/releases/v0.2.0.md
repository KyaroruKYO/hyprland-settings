# Hyprland Settings v0.2.0

Runtime-first live preview and gated single-write persistence for the
v0.55.2 data/model, audited at zero drift against Hyprland 0.55.4.

## Highlights since v0.1.0

- **Live preview without config writes**: all 341 scalar rows carry honest
  per-row classifications; 135 preview instantly (62 direct, 73 throttled)
  with revert/cancel; risky rows run under a dead-man countdown that
  auto-reverts unless kept. 36 input/cursor rows were promoted by live
  proofs on real hardware.
- **Safe Live Save Mode**: the app disables Hyprland's config autoreload at
  runtime (no file write, instantly reversible) so saving cannot reload
  your compositor mid-session. Every production Save requires the mode and
  verifies it live.
- **Gated Save**: one atomic write with a byte-exact backup, reread
  verification through the parser, and automatic backup restore on
  verification failure.
- **Save as default for Safe Live Save Mode**: persist
  `misc:disable_autoreload = true` through the same gated Save — your
  choice, never automatic.
- **Structured-family record picker**: modify existing animation records —
  the enabled flag, speed, and bezier reference (existing curves only) —
  and bezier curves (all four control points), with supervised live
  preview and gated Save. Every editable field shape carries a passed
  live-proof receipt; records and fields that cannot be safely edited say
  why. Creating and deleting records stays blocked. The style field is not
  editable (no trusted evidence for valid values) and is preserved on save.
- **Hyprland 0.55.4 compatibility**: audited at zero drift against a
  trusted `hyprctl -j descriptions` export from the official binary, with a
  repeatable read-only refresh workflow.

## Release verification

- Full validation suite green (fmt, check, 263+ test binaries, report JSON
  validity, release build, AppStream + desktop-file validation).
- v0.2.0-rc.1 was tagged and its **packaged binary passed a manual test
  pass against a real Hyprland session** (rendering, real readback,
  supervised preview with recovery including the new enabled-shape
  control, dead-man timeout auto-revert, Safe Live Save Mode runtime gate;
  zero residue, active config untouched). Details:
  `docs/MANUAL-RC-TEST-v0.2.0-rc.1.md`.

## Known limitations

- 18 touch-family rows and 3 secondary-device rows stay unproven pending
  real hardware; they are disarmed, not broken.
- monitor/bind/gesture/device/permission families stay blocked for editing
  with honest reasons.
- The animation style field is not editable; styled records are save-only
  for preview purposes.
- While an animation record is disabled, the compositor resets its
  speed/bezier readback (live-proof finding); previews that leave a record
  disabled verify the enabled flag only.
- The scalar model projects flat `section:key = value` syntax (what the
  app writes); block-syntax values in hand-written configs are not shown
  as current values.
- The data model tracks v0.55.2; 0.55.4 was audited at zero drift.
